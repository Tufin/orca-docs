The screen displays the cloud service provider accounts is currently monitoring, and their credentials. The information is separated into and .

Each line under Accounts represents one cloud vendor account and displays account information with an indication of the synchronization status for that account.

Each line under Credentials represents one set of user credentials that can assume a role to access one or more accounts under Accounts as an alternative to defining credentials at the account level.

You must define appropriate permissions in your cloud accounts to allow to monitor them. See AWS Permissions, Azure Permissions, Permissions.

Add an Account to

The required account must already exist on the cloud environment and have the required permissions.

  1. Make sure is selected in the menu.

  2. Click Add Account. The Choose Vendor Window appears.

  3. Select AWS.

  4. Enter Account Display Name. This is for your convenience only and can be any value you choose.

  5. Set up the connection method.

    There are three different ways you can give access to your AWS account:

    • Use the credentials of a user in the account
    • Give access to a user in another account via a role
    • Give access to 's own AWS account, via a role

    To use the credentials of a user in the account:

    1. Select Integration Method = Credentials .
    2. Enter Access key ID and Secret Access Key for the designated user in your AWS account.

    To give access to a user in another account via a role:

    1. Select Integration Method = Cross-Account Credentials
    2. Enter the ARN Role (Amazon Resource Name) of the designated role in the target account that can be assumed by an authorized user from your other AWS account. Example: arn:aws:iam::006755140343:role/AssumRoleAdmin.
    3. Enter the external ID if specified in the role permissions for the account that will use the role.
    4. Select the Credential Name of the account that will use the role. This must be first saved - see Add a Cross-Account Credential.

    To give access to 's own AWS account, via a role:

    1. Select Integration Method = Use Tufin Account.
    2. Enter the desired ARN Role Name. Example: arn:aws:iam::006755140343:role/AssumRoleAdmin.
    3. Enter the external ID if specified in the role permissions for the account.
  6. Click Add to save.

Add a Account to

The required subscription must already exist on the cloud environment and have the required permissions.

  1. Make sure is selected in the menu.

  2. Click Add Account. The Choose Vendor Window appears.

  3. Select Azure.

  4. Enter Account Display Name. This is for your convenience only and can be any value you choose

  5. Set up the connection method

    There are two different ways you can specify the credentials needed by to access your Azure subscription:

    1. Enter the application credentials
    2. Specify previously saved credentials

    To enter the application credentials

    1. Select Connection Method = Credentials .
    2. Enter your Azure Subscription ID.
    3. Enter Tenant from your Azure active directory.
    4. Enter Application ID from your application registration.
    5. Enter Application Secret - a client secret from your application registration

    To specify previously saved credentials:

    1. Select Connection Method = Cross-Account Credentials .
    2. Enter your Azure Subscription ID.
    3. Enter Tenant from your Azure active directory.
    4. Select the desired Credential Name from the credentials you have saved previously - see Add a Cross-Account Credential.
  6. Click Add to save.

    To enter the credentials of your service account :

Sync an Account

The sync function brings up-to-date cloud environment information for a specific account into .

  1. Click on options for the you want to sync.
  2. Select Sync account from the pop-up menu.
  3. The status tag icon on the account line will change to a syncing icon and once completed, the OK icon will be displayed. See Add an Account for more information.

    • Select a json file on your device that contains the service account credentials.

Remove a Credential

  1. Click options for the credential you want to remove.
  2. Select Remove credentials from the pop-up menu.
