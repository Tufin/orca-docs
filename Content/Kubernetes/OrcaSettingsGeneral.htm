General Settings

The Settings screen displays configuration information and lets you make changes such as switching the policy mode.

  • View the installation command and status
  • Switch the policy mode
  • Set 'Allow All' rule in Kubernetes network policies

Installation Command and Status

The version number is shown along with a status icon all good, important information or warning. An information or warning message may be displayed with a suggested action.

View the command line string needed for deploying, upgrading, restarting and uninstalling on your cluster by clicking Copy installation script.

Copy the command to the clipboard by clicking the copy icon that appears when hovering over the text and close the window when done. Before running the command on your cluster, review the optional parameters.

Policy Mode

Click on the desired mode to switch between policy modes - Enforce mode and Learn Mode.

Kubernetes Network Policies

can prevent the Kubernetes network policies from blocking traffic and interfering with the learning process. This is controlled by the Set Kubernetes Network Policy to Allow-All in Learn Mode using the toggle switch.

Enable or disable Set Kubernetes Network Policy to Allow-All in Learn Mode using the toggle switch. When set to enabled and Policy Mode is set to Learn, automatically creates an Any-Any-Allow Kubernetes network policy that allows all traffic, for each namespace. The purpose of this is to permit all traffic through the cluster and facilitate learning. An appropriate rule will be added for any new namespaces created while set to Learn. These -created rules will be removed if the appropriate namespace is removed or when Policy Mode is set to Enforce. No other Kubernetes network rules will be touched.
