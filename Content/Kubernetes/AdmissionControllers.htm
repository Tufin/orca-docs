Admission Controllers

are plugins meant to improve security of production workloads. Some 30 are shipped as an intergral part of and others can be added. They intercept authenticated API requests and execute processes when resources are created, updated, or deleted in the cluster. One example of a built-in admission controller is PodSecurityPolicy which allows you to set pod security policies and enforce a security configuration across a namespace. Unknown or outdated however, could pose a security threat if not disabled or removed.
